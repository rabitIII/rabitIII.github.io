2025/6/9 env config

Go language Linux 环境安装与配置

Linux

官网教程安装

官网安装配置文档： https://go.dev/doc/install
安装包： https://go.dev/dl/

  1. 将安装包放置到 /home/yourname 下，解压到对应的 /usr/local 目录下（默认使用 root 权限）：
rm -rf /usr/local/go && tar -C /usr/local -xzf go1.24.4.linux-amd64.tar.gz
  1. 将 golang 的环境添加到系统变量中：

好处：全局使用go指令，不需要切换到go的安装环境下使用指令；防止个人的源码污染 go 环境。

export PATH=$PATH:/usr/local/go/bin
  1. 验证 go 的全局使用(root用户 和 普通用户)：
go version

解决重启 or 重登 后的 golang 失效

参考文章: https://blog.csdn.net/super_xyo/article/details/128210788

配置环境变量

  1. 使用 vim 打开并配置 /etc/profile(不会用vim or vi的请自行百度):

注意：是 /etc 目录下 profile 文件 或者 profile.d 文件,
无关是普通用户，root用户；并且切换用户并是 cd ~ 时本质上是切换到对应用户下的根路径,
而非 OS 根路径， /etc 是 OS 根路径下的一个目录（文件夹）

添加以下命令并保存：

# golang env config

export GOROOT=/usr/local/go
export PATH=$PATH:$GOROOT/bin:$GOPATH/bin
  1. 重新加载 profile 文件使环境变量生效：
source /etc/profile
  1. 切换不同用户验证go指令 (以及退出重登验证)：
go version

代理

由于国内 长城防火墙 的原因，开发者访问外网获取对应的开发依赖普遍存在下载缓慢或者下载失败的情况，
这个时候就需要添加一些第三方代理的URI，如阿里、京东、腾讯等代理商，下载所需的依赖包。
这些代理本质上是将对应的依赖包下载保存至代理服务器上，让国内的开发者到对应的代理服务器上找到并下载依赖。

添加指令：

go env -w GOPROXY=https://goproxy.cn,direct

检查 go 环境变量：

go env
